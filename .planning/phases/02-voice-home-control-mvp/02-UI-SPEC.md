## Spacing Scale

Declared values (multiples of 4 only). Applied via `Dp` literals in Compose; extract to a `Spacing` object in `core/ui` for reuse across features.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon-to-label gaps, chip internal padding |
| sm | 8dp | Compact element spacing, badge padding |
| md | 16dp | Default element and screen edge padding |
| lg | 24dp | Section padding, card internal padding |
| xl | 32dp | Layout gaps between major UI sections |
| 2xl | 48dp | Vertical breathing room above/below mic button |
| 3xl | 64dp | Page-level top/bottom inset on VoiceScreen |

Exceptions:
- Mic button minimum touch target: 56dp (Material 3 FAB minimum; satisfies 48dp accessibility floor with visual breathing room)
- Bottom NavigationBar item touch targets: 48dp tall (Material 3 NavigationBar default; do not reduce)
- Chat bubble max width: 72% of screen width (standard chat pattern)

## Typography

Four roles only. Weights: 400 (normal) and 600 (semibold). No other weights. Defined as a custom `Typography` passed to `MaterialTheme`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16sp | 400 | 1.5 | Chat bubble message text, settings field values, reminder list items |
| Label | 14sp | 400 | 1.4 | Timestamps, source-tier badge text (`deterministic`, `unknown`), nav bar labels, input hints |
| Heading | 20sp | 600 | 1.2 | Screen titles (Chat, Home Control, Tasks, Settings), section headers |
| Display | 28sp | 600 | 1.1 | Last-reply card text on VoiceScreen (the one prominent text element in the voice UI) |

**Source:** Material 3 type scale conventions; CONTEXT.md specifies "last assistant reply shown as text below the mic button" as the primary visual anchor on VoiceScreen — assigned Display role.

## Color

Dark color scheme. A voice-first companion app runs in low-light and ambient environments; dark surfaces reduce fatigue and visually foreground the animated mic button accent.

Colors defined as a `ColorScheme` via `darkColorScheme()` in AICompanionTheme.

| Role | Value | Material 3 Role | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `#111318` | `background` / `surface` | Screen backgrounds, NavigationBar background, BottomSheet scrim |
| Secondary (30%) | `#1E2029` | `surfaceContainer` / `surfaceVariant` | Chat bubbles (assistant, outlined), cards, SettingsScreen section groups, HomeControl audit log rows |
| Accent (10%) | `#4F8EF7` | `primary` | Mic button (Idle state fill), active NavigationBar indicator, source-tier chip outline (deterministic), primary CTA buttons |
| On-Accent | `#FFFFFF` | `onPrimary` | Icon and text rendered on accent-colored surfaces |
| On-Surface | `#E2E4EC` | `onSurface` | Body text, headings, icon fills on dark surfaces |
| On-Surface-Variant | `#8A8FA8` | `onSurfaceVariant` | Timestamps, secondary labels, placeholder text |
| Destructive | `#E05353` | `error` | Delete reminder confirmation, error state mic button tint |
| On-Destructive | `#FFFFFF` | `onError` | Text/icon on destructive-colored surfaces |

Accent (`#4F8EF7`) is reserved for:
1. Mic button fill in Idle state
2. Active NavigationBar destination indicator
3. Source-tier chip outline on `deterministic` chat messages
4. Primary CTA buttons (e.g., "Save" in Settings, "Add Alias" in HomeControlScreen)

Accent is NOT used for: general text, timestamps, disabled states, secondary actions, NavigationBar icons (use `onSurface` for unselected nav icons).

**Source:** CONTEXT.md "use Material 3 color roles" (Claude's Discretion); dark scheme chosen for voice-first ambient use context.

## Voice State Color Mapping

The mic button is the primary interactive element on VoiceScreen. Its visual state MUST be unambiguous. These mappings derive from the color contract above and the CONTEXT.md state machine.

| Voice State | Mic Button Appearance | VOIC-02 |
|------------|----------------------|---------|
| Idle | Accent fill (`#4F8EF7`), mic icon white | Static |
| Listening | Accent fill with pulse animation (scale 1.0→1.12→1.0, 800ms loop) | Pulse |
| Transcribing | `surfaceVariant` fill, circular indeterminate progress indicator overlaid | Spinner |
| Processing | `surfaceVariant` fill, circular indeterminate progress indicator overlaid | Spinner |
| Speaking | Accent fill with audio-waveform bars animation (3 bars, amplitude-driven timing) | Waveform |
| Error | Destructive fill (`#E05353`), mic icon white | Static (2s, then Idle) |

**Source:** CONTEXT.md "Animated mic button for all states (VOIC-02): Single button that visually transforms per state."

## Screen-by-Screen Interaction Contract

### VoiceScreen (Start Destination)

Layout: Full-screen dark surface. Vertically centered content: mic button (56dp FAB-sized, rounded circle), 24dp gap, last-reply card below.

- **Mic button:** 56dp diameter circle. Tap once = start listening (toggle model). Tap during Speaking = stop TTS, return to Idle.
- **Last-reply card:** `surfaceContainer` background, `md` (16dp) internal padding, `lg` (24dp) corner radius. Displays most recent assistant reply in Display typography (28sp/600). Single line of text, ellipsized if overflow. Hidden when no reply exists yet.
- **No status text label beneath the button.** State is communicated through button animation alone.
- **Error toast:** Android-native `Toast.LENGTH_SHORT`. Not a persistent card. Fired from VoiceScreen's UiEffect observer.

### ChatScreen

Layout: `LazyColumn` filling screen width. Screen edge padding: `md` (16dp) horizontal. Auto-scrolls to bottom on new message.

- **User message bubbles:** Right-aligned. Accent fill (`#4F8EF7`), white text. `md` (16dp) padding inside. `lg` (24dp) corner radius, bottom-right corner `xs` (4dp) — standard chat notch convention.
- **Assistant message bubbles:** Left-aligned. `surfaceVariant` fill, `onSurface` text. Same padding/radius. Bottom-left corner `xs` (4dp).
- **Timestamp:** Label typography (14sp/400), `onSurfaceVariant` color, below each bubble, aligned to bubble edge.
- **Source-tier chip:** Below assistant bubble, above timestamp. `deterministic` = outlined chip, accent color border, Label typography. `unknown` = outlined chip, `onSurfaceVariant` border. No filled chip in Phase 2. Phase 3 adds `local-ai` and `cloud` values.
- **No date/session grouping headers in Phase 2.**

### HomeControlScreen

Layout: Scrollable `LazyColumn` with a section header "Recent Commands" (Heading typography). Optionally a second section "Device Aliases" (inline or sub-screen — planner's discretion per CONTEXT.md).

- **Audit log row:** `surfaceContainer` card, `md` (16dp) internal padding, `sm` (8dp) vertical gap between rows. Each row: command text (Body/16sp), timestamp (Label/14sp right-aligned), status icon (checkmark for success in `#4CAF50`, X for failure in Destructive `#E05353`).
- **Success green:** `#4CAF50` used only for audit log success indicator. This is NOT in the 60/30/10 palette — it is a semantic-only status color.
- **Alias editor rows:** text field + delete icon. Delete triggers destructive confirmation (see Copywriting section).

### TasksScreen

Layout: `LazyColumn` of pending reminders. Section header "Upcoming Reminders" (Heading typography). Empty state when no reminders pending.

- **Reminder row:** `surfaceContainer` card, `md` padding. Reminder description (Body/16sp), scheduled time (Label/14sp, `onSurfaceVariant`), delete icon button (Destructive color, 48dp touch target).
- **Delete:** Inline swipe-to-dismiss OR icon tap → confirmation dialog. Either implementation is valid; executor chooses based on Compose swipe-dismiss API availability.

### SettingsScreen

Layout: Scrollable `Column`. No `LazyColumn` (settings content is finite). Section headers use Heading typography with `md` top padding separator.

Sections and fields:
1. **Home Assistant** — HA Server URL (outlined text field), HA Access Token (outlined text field, `passwordVisualTransformation`, trailing show/hide toggle icon)
2. **Voice** — TTS Voice (dropdown/exposed dropdown menu of system voices)
3. **Privacy** — Privacy Mode (Material 3 `Switch`, label "Block cloud AI and cloud TTS")
4. **Automation** — Background Automation (Material 3 `Switch`, label "Enable morning briefings and scheduled automations")

- All text fields: `OutlinedTextField` from Material 3.
- All toggles: Material 3 `Switch` with accent color track when checked.
- Save action: "Save Settings" button at screen bottom, accent-filled, full width minus `md` margins.

### Bottom NavigationBar

Material 3 `NavigationBar`. 5 destinations in order:

| Position | Label | Icon |
|----------|-------|------|
| 1 | Voice | `Mic` (filled when selected, outlined when not) |
| 2 | Chat | `Chat` |
| 3 | Home | `Home` |
| 4 | Tasks | `Alarm` |
| 5 | Settings | `Settings` |

- Selected destination: accent-colored indicator pill behind icon, accent-colored label.
- Unselected: `onSurfaceVariant` icon and label.
- NavigationBar background: `background` (`#111318`).

## Accessibility Contract

- All interactive elements: minimum 48dp touch target (NavigationBar, mic button at 56dp, icon buttons padded to 48dp)
- Color is never the sole differentiator: mic state communicated by animation shape (pulse / spinner / waveform) in addition to color
- `contentDescription` required on mic button for every state: "Start listening", "Stop listening", "Processing", "Speaking — tap to stop", "Error"
- `contentDescription` on NavigationBar items: full label text ("Voice", "Chat", "Home Control", "Tasks", "Settings")
- Error toasts supplemented by vibration on error state (system haptic, no custom pattern required)
